# 🎮 터치식 리듬게임 (Touch Rhythm Game)



## 📅 프로젝트 정보
- **진행 기간**: 2024.04.19 ~ 2024.05.05
- **참여 조직**: 과학회 동아리 ’로보틱스’
- **기술 스택**: `Python (Pygame)`, `Arduino (C++)`, `Serial Communication`

---

## 📝 프로젝트 개요
과학회 동아리 활동의 일환으로 진행된 이 프로젝트는 사용자의 터치 입력을 아두이노 하드웨어로 감지하고, 이를 파이썬 기반의 게임 로직과 연동하여 동작하는 **터치식 리듬게임**을 구현하는 것이 목표였습니다.

## 🔑 주요 기능 및 내용
### 1. 통합 시스템 설계
- **하드웨어(Arduino)**: 금속판을 활용한 터치 센서 구현. 임계전압(Vth)을 설정하여 전압 변화량에 따라 터치 여부를 판별하는 아날로그 신호 처리 로직 설계.
- **소프트웨어(Pygame)**: 리듬 게임의 노트 생성, 판정, 점수 집계, 음악 재생 등 핵심 게임 루프 구현.
- **인터페이스**: Serial 통신을 통해 아두이노의 센서 데이터를 파이썬 게임 클라이트로 실시간 전송.

### 2. 기술적 구현 상세
- 아두이노의 ADC(Analog-to-Digital Converter) 데이터를 활용한 노이즈 필터링.
- Pygame 라이브러리를 활용한 그래픽 렌더링 및 키 입력 이벤트 처리.

## 🚀 직면한 어려움 및 해결 (Troubleshooting)
### 1. 터치 감도 조절 (Threshhold Tuning)
- **문제**: 환경(습도, 손의 상태)에 따라 금속판의 전도도가 달라져 터치 인식이 불안정한 문제 발생.
- **해결**: 실험을 통해 최적의 임계전압(Vth) 값을 찾고, 소프트웨어적으로 노이즈를 무시하는 데드존(Dead-zone) 로직 추가.

### 2. 통신 지연 (Latency)
- **문제**: 아두이노에서 PC로 데이터 전송 시 미세한 딜레이로 인해 리듬 게임의 판정 타이밍이 어긋나는 현상.
- **해결**: Serial 통신 속도(Baudrate)를 115200으로 상향 조정하고, 전송 데이터 패킷을 최소화하여 지연 시간 단축.

## 📚 배운점 및 성과
- **Python 역량 강화**: 단순 스크립팅을 넘어 객체 지향적인 게임 로직 설계 경험.
- **임베디드 통신 이해**: 이기종 디바이스(MCU ↔ PC) 간의 직렬 통신 원리 및 데이터 파싱 기법 습득.
- **디버깅 능력**: 하드웨어 신호와 소프트웨어 로직 양쪽을 동시에 디버깅하며 시스템 통합 역량 배양.
