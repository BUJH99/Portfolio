# 🚙 C기반 AGV 활용 무인 주차장 시스템 (AGV Automated Parking System)

> **"제약 사항을 딛고 일어선 도전: C언어로 구현한 다개체 경로 탐색 시뮬레이션"**

## 📅 프로젝트 정보
- **진행 기간**: 2025.03.01 ~ 2025.12.05
- **수 상**: 🏆 **형남과학상 동상**
- **기술 스택**: `C Language`, `Pathfinding Algorithm (A*)`, `Simulation`, `System Architecture`

---

## 📝 프로젝트 개요
AGV(Automated Guided Vehicle)를 이용한 스마트 무인 주차장 시스템을 시뮬레이션으로 구현했습니다. 객체지향 언어(C++) 대신 **C언어**만을 사용하여 자료구조와 알고리즘을 바닥부터 직접 구현하며 CS 기초를 다졌습니다.

## 🔑 주요 기능 및 내용
### 1. 다개체 제어 시스템 (Multi-Agent System)
- 여러 대의 AGV가 동시에 움직이는 환경에서 충돌 없이 목적지(주차 공간)까지 이동하는 로직 구현.
- 맵 데이터를 그리드 형태로 추상화하고, 각 AGV의 상태(위치, 적재 유무, 배터리 등)를 구조체로 관리.

### 2. 경로 탐색 및 충돌 회피
- 최단 경로 탐색을 위한 알고리즘 적용 및 테스트.
- 교착 상태(Deadlock) 방지를 위한 물리적 제약 조건 및 예외 처리 로직 추가.

---

## 🚀 직면한 어려움 및 해결 (Troubleshooting)
### 1. 언어 및 도구의 제약 (Language Constraints)
- **어려움**: C++ STL(Vector, Queue) 없이 동적 할당과 자료구조를 C언어로 직접 구현해야 하는 부담로 인한 개발 속도 저하.
- **해결**: Linked List 및 Custom Queue를 직접 구현하여 메모리 관리에 대한 깊은 이해를 얻음.

### 2. 경로 탐색 및 충돌 회피 (MAPF Challenge)
- **어려움**: 다수의 로봇이 좁은 공간에서 이동할 때 발생하는 충돌 및 데드락(Deadlock) 문제. 초기에는 단순 예외처리(If-else)로 대응했으나 코드가 비대해지고 연산량이 급증함.
- **해결**: Multi-Agent Path Finding(MAPF) 이론을 학습하려 노력하였으나 시간상 제약으로 인해 우선순위 기반의 대기(Wait) 로직과 우회 경로 재탐색 기능을 도입하여 일부 해결.

### 3. 물리적 제약 모델링
- **어려움**: 실제 물리 환경(가감속, 회전 반경)을 시뮬레이터에 반영하려다 보니 로직이 복잡해지고 교착 상태가 빈번해짐.
- **해결**: 그리드 기반의 이산적 이동 모델로 단순화하여 알고리즘의 정확성 검증에 집중함.

## 📚 배운점 및 성과
- **CS 기초 체력 증진**: 알고리즘 성능 지표(Time/Space Complexity)를 설계하고 측정하며 컴퓨터 과학적 사고력 향상.
- **시스템 최적화 경험**: 비효율적인 연산 과정을 프로파일링하고 개선하는 과정에서 성능 최적화의 중요성 체감.
- **프로젝트 관리의 교훈**: 기술적 부채(단순 예외처리의 반복)가 프로젝트 후반에 미치는 악영향을 경험하고, 초기 아키텍처 설계의 중요성을 깨달음.
