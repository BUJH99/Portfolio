# ⚙️ 레지스터 기반 MCU 제어 (Register-based MCU Control)

> **"라이브러리 없는 야생의 임베디드: 데이터시트 분석부터 펌웨어 구현까지"**

## 📅 프로젝트 정보
- **진행 기간**: 2024.09.01 ~ 2024.11.22
- **프로젝트 성격**: 레지스터 기반 MCU 제어 공모전 참가작
- **기술 스택**: `AVR/ARM MCU`, `Embedded C`, `Datasheet Analysis`, `Reverse Engineering`

---

## 📝 프로젝트 개요
아두이노의 편리한 라이브러리를 배제하고, **데이터시트(Datasheet)** 만을 의존하여 MCU의 레지스터를 직접 제어하는 Bare-metal 펌웨어 개발 프로젝트입니다. 센서, 모터, LED 등 다양한 주변장치를 하위 레벨에서 제어하며 임베디드 시스템의 본질을 학습했습니다.

## 🔑 주요 기능 및 담당 역할
### 1. Low-Level Firmware 개발
- 라이브러리 없이 레지스터 비트 조작을 통해 GPIO, PWM, Timer/Counter 제어.
- **담당 역할**: 모터 속도/방향 제어, 외부 인터럽트 서비스 루틴(ISR) 구현, 상태 표시 LED 제어 로직 작성.

### 2. 시스템 통합 및 최적화
- **폴링(Polling) vs 인터럽트(Interrupt)**: 초기 폴링 방식의 한계를 체감하고, 실시간성을 보장하기 위해 인터럽트 기반 스케줄링으로 구조 변경.
- **역공학(Reverse Engineering)**: 본선 과제로 시중 전자제품의 신호를 오실로스코프로 분석하여 통신 프로토콜을 파악하고, 이를 MCU로 재구현.

---

## 🚀 직면한 어려움 및 해결 (Troubleshooting)
### 1. 폴링과 인터럽트의 구조적 차이
- **어려움**: 단일 루프에서 폴링 방식으로 여러 센서를 처리하다 보니 반응 속도가 느려지고 데이터 유실 발생.
- **해결**: 긴급한 이벤트(센서 감지)는 인터럽트로 처리하고, 덜 중요한 작업은 메인 루프에서 처리하도록 펌웨어 아키텍처 개선.

### 2. 하드웨어 자원 제약 (Resource Constraints)
- **어려움**: MCU의 물리적 핀 개수 부족 및 제한된 전력 공급으로 인한 오작동.
- **해결**: 핀 멀티플렉싱(Multiplexing) 기법 활용 및 회로 간소화를 통해 자원 효율성 극대화.

### 3. 클럭(Clock) 동기화
- **어려움**: 부정확한 타이밍으로 인한 통신 에러.
- **해결**: 데이터시트의 Clock Tree를 정밀 분석하여 Prescaler를 재설정하고 타이밍 마진 확보.

## 📚 배운점 및 성과
- **Datasheet 리터러시**: 수백 페이지의 기술 문서를 독해하고 필요한 레지스터 정보를 추출하여 코드에 적용하는 능력 배양.
- **임베디드 기초 확립**: 추상화된 라이브러리 뒤에 숨겨진 MCU의 실제 동작 원리(Interrupt Vector Table, Memory Map 등) 체득.
- **문제 해결 능력**: 오실로스코프 등 계측 장비를 활용한 하드웨어 디버깅 능력 향상.
